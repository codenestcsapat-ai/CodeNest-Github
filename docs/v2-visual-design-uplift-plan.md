# CodeNest V2 Visual Design Uplift Plan

## Current Gap Summary

The V2 preview is functional and content-correct, but the visual system still needed more craft. The biggest gap was that sections felt like separate stacked blocks instead of one calm, polished landing page. Project and case study screenshots were available, but the surrounding UI needed stronger framing, better rhythm, and more confidence.

## Target Visual Principles

- Warm off-white paper background with subtle sand and green layers.
- Dark navy typography with calm hierarchy and readable Hungarian line breaks.
- Muted green as the primary action and trust accent.
- Cards should feel related, soft, and intentional, not identical boxes repeated everywhere.
- Mockups and screenshots should communicate usable web systems: public site, admin logic, mobile view, content rows, and workflow hints.
- The studio should feel small, direct, human, and professional.

## What Was Improved In This Pass

- Added a controlled CSS uplift layer for section rhythm, card polish, shadows, border softness, and background texture.
- Refined the homepage hero balance and made the mockup composition feel more integrated.
- Reduced the boxed-section feeling by letting section shells breathe on the page background.
- Made service cards feel more like three concrete offers with quieter density and cleaner module hints.
- Strengthened the Gárdony featured project area and supporting project screenshot cards.
- Added a more contact-like preview structure with trust notes, project type choices, field placeholders, and a clear conversation CTA.
- Added a screenshot-led case study hero card so each case study feels more reference-like from the first screen.
- Polished the deep navy footer transition and responsive behavior for smaller screens.

## What Remains For Later

- Real browser review should decide whether the hero mockup needs further simplification on small mobile screens.
- Supporting project screenshots may still need recapture if any source image is too low resolution or awkwardly cropped.
- Case studies are still intentionally short reference pages; richer project storytelling can come later.
- The contact area is still a visual preview and is not wired to a form service in this sprint.
- No live homepage migration has been done.

## Future Pages Not Implemented Yet

- Service detail pages.
- Works listing page.
- Dedicated contact page.
- Legal pages refresh.
- 404 page.
- Full design system and style guide.

## Continuation Pass Notes

This follow-up pass kept the existing V2 structure and focused on visual refinement only.

- The hero received tighter headline scale, clearer CTA rhythm, and a more integrated public/admin/mobile mockup composition.
- Service cards received stronger hierarchy, calmer density, and more crafted module hints.
- Project proof cards received stronger framing, cleaner thumbnail treatment, and a more reference-like Gárdony feature composition.
- Case study pages received additional screenshot-led polish for the hero, overview, visual section, and bottom CTA.
- Contact and footer styling were refined to feel more conversational, trustworthy, and structured.

No live homepage migration, SEO/AEO implementation, i18n work, or contact integration was included in this continuation pass.
